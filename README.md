- 데이터

본 프로젝트는 2025년 외국인 방문객의 특성과 입국 흐름을 분석하기 위해 서울 열린데이터광장에서 제공하는 전국 기준 외국인 방문객 통계를 활용하였다. 분석에 사용한 데이터는 다음 세 가지이다.

첫째, 연령별 외국인 방문객 데이터를 활용하였다. 이 데이터는 외국인 방문객을 대륙별로 구분하고, 각 대륙별 방문객 수를 연령대별로 제공한다. 본 프로젝트에서는 전체 방문객 수와 20대·30대 방문객 비중을 함께 분석하여 외국인 방문객의 연령 특성을 파악하였다.

둘째, 입국목적별 외국인 방문객 데이터를 활용하였다. 이 데이터는 외국인 방문객의 입국 목적을 관광, 상용, 공용, 유학/연수, 기타로 구분한다. 이를 통해 대륙별 방문 목적의 차이와 관광 목적 방문객 비중을 분석하였다.

셋째, 입국교통수단별 외국인 방문객 데이터를 활용하였다. 이 데이터는 외국인 방문객이 공항 또는 항구를 통해 입국한 규모를 제공하며, 세부적으로 인천공항, 김해공항, 김포공항, 제주공항, 부산항, 인천항, 제주항 등 입국 지점별 방문객 수를 포함한다. 이를 통해 외국인 방문객의 주요 입국 경로와 공항 의존도를 분석하였다.

분석을 위해 SQLite 데이터베이스 visitor.db를 구축하였으며, 데이터베이스에는 age, purpose, transport 세 개의 테이블을 생성하였다.

- 데이터 클렌징

원본 데이터는 공공데이터에서 제공된 엑셀 파일을 기반으로 하였으며, 분석 및 데이터베이스 적재를 위해 CSV 파일로 변환하였다. 이후 Python과 SQLite에서 활용하기 쉽도록 데이터 구조를 정리하였다.

먼저, 원본 데이터에서 분석에 필요한 행과 컬럼만 남기고 불필요한 제목 행, 설명 행, 중복 헤더를 제거하였다. 모든 데이터는 대륙별 집계 구조를 가지고 있었기 때문에 대륙 컬럼을 기준으로 정리하였다. 전체 합계 행은 소계로 유지하여 전체 방문객 기준의 분석에 활용하였다.

다음으로, SQL에서 계산이 가능하도록 방문객 수 관련 컬럼을 정수형 데이터로 정리하였다. 예를 들어 전체 방문객 수, 연령대별 방문객 수, 입국 목적별 방문객 수, 공항·항구별 입국자 수는 모두 숫자형으로 처리하였다. 원본 데이터에서 결측 또는 해당 없음에 가까운 값은 계산 오류가 발생하지 않도록 0으로 처리하였다.

또한 세 데이터는 모두 2025년 외국인 방문객을 대륙 기준으로 집계한 독립 통계 자료이므로, 외래키는 설정하지 않았다. age, purpose, transport 테이블은 부모-자식 관계가 아니라 각각 연령, 목적, 교통수단이라는 서로 다른 기준으로 집계된 병렬 테이블이다. 따라서 필요할 때만 대륙 컬럼을 기준으로 비교하였다.

마지막으로, 대시보드에서는 원본 CSV의 wide format 구조를 유지하였다. 즉, 연령대, 입국 목적, 입국 지점이 각각 별도의 컬럼으로 존재하는 형태를 사용하였다. Streamlit 시각화 과정에서는 필요한 경우 Python에서 일부 컬럼을 차트용 데이터프레임으로 변환하여 사용하였다.
